I ported the BSV SDK to Zig.

The TypeScript SDK (@bsv/sdk on npm) and the Go SDK (bsv-blockchain/go-sdk) are the two reference implementations for building on BSV. I took both and rebuilt the full surface in Zig from scratch — pure Zig stdlib, no vendored C, no OpenSSL, no libsecp256k1.

It started because I wanted to learn Zig. I had solid reference code in two languages, BSV has zero Zig ecosystem, and Runar had just come out. A full SDK port seemed like an absurd goal by last year's standards, but it turned out to be very achievable with today's tooling.

What's in it

The library covers 27 BRC standards. The script interpreter passes 1,435 of Go's 1,499 test vectors (the remaining 64 are metadata rows). Script-only workloads benchmark 10-40x faster than the Go interpreter on Apple Silicon, while P2PKH verification lands at roughly the same speed since secp256k1 point math dominates both.

Keys and crypto:
- secp256k1 ECDSA, Schnorr proofs, ECDH, Type-42 key derivation
- BIP32 HD keys, BIP39 mnemonics, Shamir secret sharing
- compact signatures with recovery
- AES-CBC, AES-GCM, ECIES (Electrum + Bitcore), HMAC-DRBG

Script:
- full execution engine with CHECKSIG, CHECKMULTISIG, post-Genesis behavior
- Go-shaped policy flags, ASM encode/decode, script builder
- type detection (isP2PKH, isData, isMultiSigOut)
- templates for P2PKH, OP_RETURN, PushDrop, R-puzzle, OP_TRUE

Transactions:
- raw and extended format parse/serialize
- BEEF V1/V2/Atomic
- transaction builder with addInput/addOutput/payToAddress/sign
- fee calculation with pluggable models, hex roundtrip, clone

SPV:
- MerklePath with computeRoot, combine, verify
- ancestor traversal, BEEF verification, pluggable chain tracker

Broadcast:
- WhatsOnChain, TAAL, Arc HTTP clients

Compat:
- Bitcoin Signed Message sign/verify/recover
- ECIES, P2PKH addresses, WIF, Base58Check

Why Zig

Zig gives you what you'd reach for C for, with memory safety you can actually reason about. Explicit allocators everywhere — you see every allocation and every free. The script interpreter runs on stack memory with zero heap allocation for simple scripts.

It compiles to a static binary with no dependencies. Cross-compile to any target from any host. Link it into C, C++, Rust, or anything with a C ABI. That's how BSV primitives get into embedded systems, WASM, game engines, or anywhere a Node runtime doesn't belong.

The secp256k1 implementation uses Zig's built-in verifier directly. The code stays pure Zig stdlib with no vendored C, OpenSSL, or libsecp256k1.
